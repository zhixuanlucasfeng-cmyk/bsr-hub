# Anna Accessibility and Mobile Checklist

- [x] Semantic navigation, main, section, article, label, heading, and button elements.
- [x] Every search, filter, persona, booking, and listing input has a visible or accessible label.
- [x] Keyboard focus uses native interactive controls; no hover-only action is required.
- [x] Text and controls retain contrast on light, dark, purple, and lime surfaces.
- [x] Layouts collapse at 1000 px and 650 px; cards become one column on phones.
- [x] Modal fits within the viewport and scrolls without hiding the close button.
- [x] Public location is city/state only; no private-address field exists.
- [x] API-offline, unavailable action, empty-order, and success notices are written in plain English.

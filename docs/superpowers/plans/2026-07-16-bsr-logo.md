# BSR Logo Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Produce a simple unified BSR brand mark plus Hub and Runner wordmarks that match the approved website palette.

**Architecture:** Use the built-in image generation workflow to create one icon-only mark and two horizontal wordmarks from the same locked visual prompt. Validate text, palette, small-size legibility, and family consistency before saving final assets in the website workspace.

**Tech Stack:** Built-in image generation, PNG raster assets, local visual inspection.

## Global Constraints

- Preserve the exact text `BSR`, `BSR Hub`, and `BSR Runner`.
- Use a deep-charcoal rounded square, white lettering, and a small lime circular-arrow accent.
- Keep the design flat, vector-friendly, modern, and readable at small sizes.
- Do not use gradients, shadows, mascots, slogans, mockup scenes, watermarks, or detailed illustrations.
- Use purple only as an optional minor accent.

### Task 1: Generate the Unified Logo Family

**Files:**
- Create: `apps/web/public/brand/bsr-icon.png`
- Create: `apps/web/public/brand/bsr-hub-logo.png`

**Interfaces:**

- [ ] **Step 1: Generate the icon-only mark**

Generate a square, centered logo with exact white `BSR` lettering inside a deep-charcoal rounded square and one small lime circular-arrow accent. Use a plain white background, generous padding, and no additional words.

- [ ] **Step 2: Generate the Hub horizontal lockup**

Generate the same icon at left with exact `BSR Hub` wordmark at right. Keep a plain white background, balanced spacing, and no other copy.

- [ ] **Step 3: Generate the Runner horizontal lockup**

Generate the same icon at left with exact `BSR Runner` wordmark at right. Keep construction, typography, scale, and spacing consistent with the Hub lockup.

- [ ] **Step 4: Inspect all three outputs**

Confirm the lettering is spelled exactly, the icon remains readable when viewed small, the arrow does not obscure `BSR`, the palette matches the website, and the two wordmarks clearly belong to one family. Reject and regenerate any asset that fails one of these checks.

- [ ] **Step 5: Save the approved assets**

Copy the three selected PNG files into `apps/web/public/brand/` using the exact filenames above. Do not overwrite any pre-existing brand asset with a different filename.

- [ ] **Step 6: Verify file integrity**

Run:

```bash

du -h apps/web/public/brand/*.png
```

Expected: three readable PNG images with non-zero file sizes.
